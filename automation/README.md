# SauceDemo Automation

Python Playwright/PyTest automation for the SauceDemo storefront (https://www.saucedemo.com/). Generated by the `agentic-playwright-automation` skill.

## Layout

```
automation/
├── pyproject.toml         PDM-managed dependencies and tool config.
├── pytest.ini             Pytest configuration and marker registry.
├── Makefile               install / lint / test / report targets.
├── .env.example           Template for local environment overrides.
├── config/                Environment configuration (environments.yaml, settings.py).
├── framework/             Page objects, components, models, data, utilities.
├── tests/                 Tests + conftest.
├── test_data/<env>/       YAML test data partitioned by environment.
├── reports/               HTML, JUnit, traces, screenshots, automation reports.
└── docs/                  Framework rules and standards.
```

See `docs/` for the rules and standards that govern how to add tests, page objects, fixtures, and test data.

## Quickstart

From this directory:

```
make install            # PDM installs dependencies
make install-browsers   # Playwright browser binaries
make test-smoke         # Run smoke tests
make test-report        # Run all tests and write reports/html/report.html
```

## Configuration

The active environment is selected by the `APP_ENV` environment variable (`local`, `dev`, or `qa`). Environment values live in `config/environments.yaml`; per-developer overrides live in a local `.env` (copy `.env.example`).

Do not hard-code URLs, credentials, or product data in tests or page objects. Pass values through fixtures.
