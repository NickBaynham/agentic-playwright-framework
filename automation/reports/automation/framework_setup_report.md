# Framework Setup Report

## Summary

Scaffolded a Python Playwright/PyTest automation framework under `automation/` targeted at the SauceDemo storefront (https://www.saucedemo.com/). The scaffold follows the structure mandated by the `agentic-playwright-automation` skill, uses PDM for package management, and ships with a single framework-smoke test that verifies pytest, the environment loader, and the test-data loader are wired together. No application tests are generated by this run; those will come from `/convert-bdd-to-playwright` and `/generate-playwright-test`.

No packages were installed and no tests were run. Run `make install && make install-browsers` to install dependencies and Playwright browser binaries when ready.

## Risks and Open Questions

- **Pinned versions vs. latest:** dependency versions were pinned to recent stable releases as of 2026-05. CLAUDE.md asks for latest APIs; bump pins via `pdm update` and verify before relying on them.
- **`headless: false` for local:** the `local` environment defaults to a visible browser, matching the demo's developer workflow. CI should override via `HEADLESS=true` or by selecting `APP_ENV=dev`/`qa`. The `tests/conftest.py` override of `browser_type_launch_args` bridges the resolved `settings.headless` into pytest-playwright; the `browser` field in `environments.yaml` is currently advisory — browser selection happens via pytest-playwright's `--browser` CLI flag.
- **Screenshot-on-failure hook is wired.** `tests/conftest.py` registers a `pytest_runtest_makereport` hook that calls `framework.utils.evidence.save_screenshot` for any failing test that consumes the `page` fixture. No-op for non-UI tests.
