# Locator Candidate Checklist

Apply to a completed session report, page observation, or standalone locator candidate document. Each item should be Pass, Fail, or Needs Review.

| Check | Status | Notes |
|---|---|---|
| Meaningful workflow elements were captured (fields, buttons, links, headings, menus, cards, badges, messages, workflow-critical controls) | | |
| A candidate Playwright locator was recorded for each element | | |
| Accessibility role and accessible name were considered | | |
| Label or placeholder was considered for input fields | | |
| Test ID or data attribute was captured if available | | |
| Repeated element risks (multiple matches, scoping) were documented | | |
| Dynamic or conditional element risks (visibility on state change, lazy loading) were documented | | |
| Confidence (High / Medium / Low) was assigned to each candidate | | |
| Rationale is included for each candidate | | |
| Locator candidates are clearly marked as non-final | | |
| Tooling notes are kept separate from locator decisions | | |
| Automation Handoff Notes section is present and populated | | |
